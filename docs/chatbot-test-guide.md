## 테스트 시나리오

### 시나리오 1: 목록 조회
1. 카카오톡에서 채널 챗봇 대화방 열기
2. `인증관리` 입력
3. **기대 결과:**
   - 카드 캐러셀 표시 (팀명 + 사진 썸네일 + 승인/거절 버튼)
   - 대기 건 없으면: `대기 중인 인증 요청이 없습니다. ✅`

### 시나리오 2: 승인 플로우
1. `인증관리` → 카드 표시
2. `✅ 승인` 버튼 클릭
3. **기대 결과:**
   - `✅ {팀이름} 팀 인증이 승인되었습니다!`
   - 하단에 `📋 인증 목록` 바로가기
4. `📋 인증 목록` 클릭 → `대기 중인 인증 요청이 없습니다. ✅`

### 시나리오 3: 거절 플로우
1. `인증관리` → 카드 표시
2. `❌ 거절` 버튼 클릭
3. **기대 결과:** `'{팀이름}' 팀의 거절 사유를 입력해주세요.`
4. 거절 사유 텍스트 입력 (예: `사진이 불분명합니다`)
5. **기대 결과:**
   - `❌ {팀이름} 팀 인증이 거절되었습니다.` + 사유 표시
   - 하단에 `📋 인증 목록` 바로가기

### 시나리오 4: 예외 케이스
| 상황 | 기대 결과 |
|------|-----------|
| 대기 건 없을 때 `인증관리` | `대기 중인 인증 요청이 없습니다. ✅` |
| 이미 처리된 건 승인 재시도 | `이미 approved 처리된 요청입니다.` |
| 거절 사유 입력 중 다른 명령어 | 컨텍스트 수명 1이라 다음 턴에서 만료, 정상 라우팅 |

## 트러블슈팅

### 카드 대신 텍스트만 나옴
→ 오픈빌더 "인증관리" 블록에서 "스킬 데이터 사용" 체크 + 스킬 "인증목록조회" 연결 확인

### 승인/거절 버튼 눌러도 반응 없음
→ 오픈빌더 발화 패턴 확인: `승인하기` / `거절하기` (기존 `승인` `거절`이면 안 됨)

### 거절 사유 입력 후 엉뚱한 응답
→ "거절확정" 블록의 입력 컨텍스트 `reject_context` 설정 확인

### 스킬 응답 시간 초과 (5초)
→ Cloud Functions 콜드스타트 문제. 한 번 더 시도하면 해결됨

### 사진 썸네일이 안 보임
→ Firebase Storage 이미지 URL이 공개 접근 가능한지 확인 (storage.rules에서 read: if true)

## 프론트엔드 변경사항 (club-detail.js)
바텀시트에서 인증 상태별 UI 분기 구현 완료:
- 신청 이력 없음 → `✅ 인증 신청` 버튼
- `pending` → `⏳ 인증 심사 중입니다` 안내
- `rejected` → 거절 사유 표시 + `🔄 인증 재신청` 버튼
- `approved` → 인증 배지 표시
