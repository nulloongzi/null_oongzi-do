# 09. 다국어(i18n) — 토글 · 사전 · 데이터 변환

> 원본: `js/i18n.js`(부트는 `js/app.js`). 시각 영향은 [design.md §12](../design.md).
> 외국인 축이 진짜 요구사항이므로 i18n 은 메뉴 번역을 넘어 **데이터 표시(가격/대상/요일)까지** 변환한다.

## 1. 언어 상태
- `currentLang ∈ {'ko','en'}`, 저장 키 `nulloong_lang`.
- 자동감지 `readLang()`: 저장값 우선 → 없으면 `navigator.language` 가 `ko*` 아니면 `'en'`, 맞으면 `'ko'`.

## 2. 사전 구조
```
DICT = { key: { ko:'한글', en:'English' }, ... }
```
키 네이밍: UI(`brand/search_ph/login...`), 클럽(`reg_*`,`cd_*`), 픽업(`pk_*`,`tab_pickup`,`pk_sport_*`,`pk_lv_*`), 도시락(`lb_*`), 요일(`d_mon..d_sun`).

## 3. 함수
| 함수 | 동작 |
|---|---|
| `t(key, fallback?)` | `DICT[key][currentLang]`, 없으면 fallback/key |
| `tf(key, params)` | `t(key)` + `{param}` 토큰 치환 |
| `setLang(lang)` | currentLang 변경, localStorage 저장, `<html lang>`/`body.lang-en` 갱신, `nurungji:langchange` 이벤트 발행 |
| `toggleLang()` | 반대 언어로 setLang |
| `applyI18n()` | DOM 일괄 번역: `[data-i18n]`→textContent, `[data-i18n-placeholder]`→placeholder, `[data-i18n-html]`→innerHTML |

## 4. DOM 속성 (웹 패턴)
- `data-i18n` : 정적 텍스트(버튼/라벨).
- `data-i18n-placeholder` : 입력 placeholder.
- `data-i18n-html` : 관리자 작성 HTML(예 등록 tip) — XSS 위험 없는 신뢰 문자열만.
- **Flutter**: 이 속성 시스템 대신, 모든 텍스트를 `t('key')` 로 조회하고 언어 상태(`ChangeNotifier`/Stream) 변화 시 rebuild. JS의 `nurungji:langchange` 리스너들(픽업 리스트/상세, 클럽 상세, 탭 chrome 재렌더)은 Flutter에서 언어 의존 위젯 자동 rebuild 로 대체.

## 5. 데이터 변환(표시 시점) — 한국어 데이터를 영어로 즉석 변환
Firestore 데이터는 **한국어로 저장**, 영어 모드에서 표시만 변환:
- `i18nTarget(str)` : 대상 어휘 치환(여성전용→Women only, 남성전용→Men only, 선출가능→Ex-players OK, 대학생→College, 청소년→Youth, 성인→Adults, …).
- `i18nPrice(str)` : "6만원"→"₩60,000", "게스트비"→"Guest fee" 등(정규식 `(\d+)\s*만\s*원?` → ₩ 천단위).
- `i18nDay(kchar)` : '월'→Mon … '일'→Sun(키 `d_mon..d_sun`). 스케줄 "토 19:00"→"Sat 19:00".

> **Flutter**: 동일 변환 유틸을 두어 **저장은 KO, 표시만 변환**. 이게 빠지면 영어 사용자에게 클럽/가격/요일이 한국어로 노출되어 외국인 축이 깨진다.

## 6. 초기화
로드 시: `<html lang>` 설정, `body.lang-en` 토글, `document.title=t('brand')`, `applyI18n()`.

## 7. 주요 픽업 키 예시(참고)
`tab_pickup`(픽업/Pickup), `pk_list_title`(여기서 픽업이 열려요/Where pickup happens), `pk_host_title`(픽업 등록/Add a pickup), `pk_sport_6s`(6인제/6s), `pk_lv_beginner`(입문/Beginner), `pk_beginner_ok`(🌱 초보환영/Beginners welcome), `pk_english_ok`(🌐 English OK), `pk_thisweek_badge`(이번주/This week), `pk_contact_cta`(💬 단톡 들어가기/Join the group chat), `pk_empty`, `pk_req_fields`.

## 8. Flutter 매핑 요약
- 사전(KO/EN) 그대로 이식(키 동일) — 추후 `.arb`/intl 로 옮겨도 키 유지.
- 언어 자동감지(기기 로케일 non-ko→en) + 수동 토글 + 영속(SharedPreferences).
- **표시 시점 데이터 변환**(target/price/day) 반드시 구현.
- 영어 레이아웃 규칙([design.md §12](../design.md)): 6s 우선/강조, 힌트 노출, 긴 라벨 한 줄 유지.
