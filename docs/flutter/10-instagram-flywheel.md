# 10. 인스타그램 친화 전략 — 발견↔공유 플라이휠

> 이 앱을 다른 배구 플랫폼(랭구랭구·배구구)과 가르는 **핵심 차별점**. 단일 기능이 아니라 제품 전체를 관통하는 원칙이다.
> 근거: [PHILOSOPHY.md](../PHILOSOPHY.md) — *"인스타와 앱은 하나의 깔때기. 릴스→앱, 앱→팔로우로 서로 먹인다."* / *"누룽지에게 DM하는 느낌."*
> 관련 구현: [07-share](./07-share.md), `js/insta-embed.js`, `js/share.js`.

## 1. 플라이휠 (이게 제품의 심장)

```
        ┌──────────────────────────────────────────────┐
        │                                                │
   ① 앱에서 발견                                    ④ 앱에 다시 표시
   (지도로 동호회/픽업 찾기)                        (그 장소의 릴스가
        │                                          상세시트에 임베드)
        ▼                                                ▲
   ② 인스타로 위치 공유                            ③ 거기서 운동한 영상을
   (스토리 카드 + 링크 스티커                       인스타에 게시
    → ?club=/?spot= 딥링크)                          (릴스/게시물)
        │                                                │
        └───────────► 인스타 (외부) ◄───────────────────┘
```

- **앱 → 인스타**: 발견한 장소를 스토리로 공유(링크 스티커로 딥링크 전파) → 팔로워가 링크 타고 앱 진입.
- **인스타 → 앱**: 그 장소에서 찍은 릴스가 클럽/픽업 상세에 임베드되어, 다음 사람이 "여기 분위기 이렇구나"를 영상으로 확인 → 다시 발견·공유.

> **가치 필터 정렬**(philosophy §가치필터): 이 루프는 *평가/랭킹이 아니라* "적합·소속"(여기 분위기 나랑 맞나)을 전달한다. 릴스는 별점의 반대 — **환대의 증거**다. Flutter에서도 영상/공유를 절대 "점수·순위"로 변질시키지 말 것.

## 2. 현재 구현 상태 (정확히 — 무엇이 자동이고 무엇이 수동인가)

| 단계 | 구현 | 상태 |
|---|---|---|
| ① 발견 | 지도·필터·검색 | ✅ 자동 |
| ② 위치→IG | 스토리 카드(1080×1920) + **링크 스티커용 딥링크**(`?club=`/`?spot=`) + QR. 네이티브 브리지 `NativeShare.postMessage({type:'ig_story', stickerImage, contentUrl, ...})`, 웹은 프리뷰+다운로드 폴백. OG 메타태그로 링크 미리보기. | ✅ 구현(앱은 네이티브 공유로) |
| ③ IG에 영상 게시 | 사용자가 인스타에서 직접 | ⛔ 앱 밖(우리 통제 밖) |
| ④ 영상→앱 표시 | `insta_reel` 필드(공개 릴스/게시물 permalink)를 **소유자가 등록/수정에서 붙여넣으면** 클럽/픽업 상세에 `renderInstaEmbed`로 임베드 | ⚠️ **수동 큐레이션** |

**핵심 사실**: ④는 자동 수집이 아니다. 클럽·픽업당 `insta_reel` **1개**를, 소유자(또는 관리자)가 등록폼/수정폼·픽업 호스트 모달의 "릴스/게시물 링크" 필드에 직접 입력한다. 임베드는 공개 blockquote + Instagram `embed.js`(cross-origin)라 **앱 내 자동재생·세로 몰입·스와이프는 불가**(공개 임베드 한계).

### 코드 위치 (인수인계용)
- 쓰기(③→④ 연결): `registration.js`(`regReel`→`insta_reel`, `sanitizeInstaPostUrl` 검증), `pickup-host.js`(`pkReel`→`insta_reel`).
- 읽기/표시(④): `club-detail.js:305`, `pickup-detail.js:122` → `insta-embed.js renderInstaEmbed(container, url)`.
- 공유(②): `share.js` `buildClubShareUrl/buildSpotShareUrl`, `shareStory`(네이티브 브리지), 스토리 카드 생성.

## 3. Flutter에서 보존·강화할 것

### 3.1 반드시 그대로 (루프가 끊기지 않게)
- **딥링크 동일**: `?club=<id>`/`?spot=<id>` 를 App Links로 받아 앱 내 상세로 라우팅. 웹과 동일 URL을 공유하므로(스토리 링크 스티커가 이 URL), 앱은 이 URL을 가로채 네이티브 상세를 연다. → [00-overview §2 딥링크](./00-overview.md), [07-share](./07-share.md).
- **스토리 카드 + 링크 스티커**: 1080×1920 카드를 네이티브로 그리고(`CustomPainter`/`RepaintBoundary`), 인스타 스토리 공유 intent에 **스티커 이미지 + `content_url`(딥링크)** 를 함께 전달. (웹의 `NativeShare` 브리지가 하던 일을 앱에선 네이티브가 직접.)
- **릴스 임베드 표시(④)**: 상세시트의 `insta_reel` 을 표시. 앱에선 (a) `WebView`로 동일 blockquote+embed.js, 또는 (b) 썸네일 카드+탭 시 인스타 앱/외부 오픈. URL 검증(`sanitizeInstaPostUrl`)은 유지.
- **OG/링크 미리보기**: 공유 링크(웹)의 OG 메타는 그대로(외부에서 링크 펼침 시 카드 노출).

### 3.2 인스타 네이티브 앱으로서 더 좋아질 여지 (제안 — 노스스타 위배 없이)
> 모두 "환대·발견" 강화 방향이어야 하고, 랭킹/지표화는 금지.

- **딥링크 정밀화**: 스토리에서 들어온 사용자는 곧장 해당 상세 + "이 근처 더 보기"로 자연 연결(앱 내 라우팅이라 웹보다 매끄럽게 가능).
- **공유 마찰 최소화**: 클럽/픽업/내 프로필(포장하기) 어디서나 1~2탭에 스토리 공유. (이미 구조는 있음 — 앱에선 네이티브 공유라 더 빠름.)
- **릴스 수집 자동화(④의 수동→반자동, 장기)**: 현재는 소유자가 permalink 1개 수동 입력. 가능 경로:
  - **해시태그/멘션 컨벤션**: 클럽이 게시 시 정해진 핸들 멘션/해시태그 → (운영자가) 큐레이션해 `insta_reel` 채움. (여전히 opt-in·수동 검수 — philosophy의 "합의" 원칙 유지.)
  - **여러 릴스**: `insta_reel`(string) → `insta_reels`(string[]) 확장 시, 상세에서 가로 캐러셀로. (스키마 변경이라 웹과 동시 합의 필요.)
  - **Instagram Graph API(B안)**: 진짜 릴스탭 몰입(세로·자동재생). 비용·심사·동의 필요 — 장기 과제. (insta-embed.js 주석에 "B(Graph API) 필요"로 명시됨.)
  - ⚠️ 어떤 자동화도 **클럽 동의(opt-in)** 없이 영상을 끌어오지 않는다(philosophy 가치필터 5번).

## 4. 디자인 함의 (요약 — 상세는 [design.md](../design.md))
- 인스타 임베드 박스는 단정하게(폭 ≤360, 가운데). 앱에선 세로 카드/캐러셀로 더 몰입감 있게 가능.
- 스토리 카드 = 브랜드 접점. 누룽지 톤(크림·옐로·따뜻함) + 딥링크 QR/스티커 일관.
- 공유 버튼은 클럽/픽업/프로필 상세에서 눈에 띄되 강요 아님(환대 톤).

## 5. 한 줄 요약 (개발자에게)
> **"발견은 앱에서, 자랑은 인스타에서, 그 자랑이 다시 앱의 환대가 된다."**
> 딥링크(`?club=`/`?spot=`)·스토리 카드(링크 스티커)·`insta_reel` 임베드 — 이 셋이 끊기면 차별점이 사라진다. Flutter에서 반드시 셋 다 살릴 것.
